# Solutions — Portfolio Map

> **Purpose**: This file is the single source of truth for projects that are part of Sameer Goel's solutions portfolio but are **not hosted inside this repo**. An AI agent building a portfolio view should treat the projects listed here as equal citizens alongside the ones in this repo.
>
> Every project here has a **category** matching the taxonomy in [`CATEGORIES.md`](CATEGORIES.md), so the portfolio can be rendered as one unified view.

## How an AI agent should read this

1. Start with `CATEGORIES.md` to understand the category system.
2. Scan `README.md` for internal projects (hosted in this repo at `<category>/<slug>/`).
3. Scan this file (`SOLUTIONS.md`) for external projects (hosted elsewhere).
4. Merge the two into a single categorized portfolio.
